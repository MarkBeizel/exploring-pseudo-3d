•Установка удалённого репозитория на локальный компьютер:
1.Если собираешься клонировать удалённый репозиторий,
не стоит использовать git init, remote add origin, pull -
- лучше писать clone - он совмещает функции трёх команд.
Если не работает, то проверить credential manager на сохранённые
логин и пароль. Если чьи-то чужие credentials и вправду
присутствуют - удалить их, запустить git clone ещё раз.

•Сохранение изменении в репозитории
1.Чтобы передать изменения на удалённый сервер нужно сначала сохранить
изменения на локальном репозитории:
$git commit -m "Сообщение об изменениях"
*Флаг -m даёт вписать сообщение в одной строке сommit (так удобнее).
 Можно не писать -m флаг и сообщение, тогда его придётся писать в vim консоле.
 Чтобы сохранить/применить сообщение и выйти из консоли нужно использовать
 комбинацию клавиш: ctrl+c, и затем вписать: :q!.
а затем использовать команду:
$git push origin. 

•Если надо переименовать последний commit на удалённый репозиторий:
1.Используй:
$git commit --amend -m "твоё изменённое сообщение"
*Опция --amend используется в Git для изменения последнего коммита. 
 Она позволяет отредактировать содержимое коммита (например, добавить или удалить файлы) 
 или изменить его сообщение. Это полезно, если нужно исправить ошибку в последнем коммите, 
 не создавая новый, а просто изменив уже существующий. Команда git commit --amend 
 перезапишет последний коммит, сохраняя изменения, и при необходимости откроет редактор
 для изменения его сообщения. -m как говорилось ранее позволяет избежать неудобную vim консоль
 и вписать сообщение сразу.
затем:
$git push origin HEAD:main -f
*HEAD:main = "локальная ветка из которой отправить изменения":"удалённая ветка"
 HEAD на лоакльном репозитории - указывает на активную ветку, 
 с которой данный момент происходит работа.
 HEAD на удалённом репозитории - это ссылка на основную ветку. 
 Она указывает на ветку, которая является основной в удаленном репозитории, 
 например, на ветку main или master, в зависимости от настроек репозитория.
*Также есть способ переименовать несколько commit сразу,
 или же просто переименовать предпоследний и любой другой,
 используют:
 $git rebase -i HEAD^

•Установка среды и сохранения списка библиотек:
1.Используй:
$python -m venv "название папки (по стандарту ".venv")"
*здесь мы просим запустить python модуль venv (флаг -m), который установит 
 виртуальную среду в наш репозиторий.
2.Можно использовать:
$pip install "название билиотеки"
*устанавливает указанную библиотеку в папку Lib в папку виртуальной среды.
*pip - is the package installer for Python.
2.Можно использовать:
$pip install -r "список библиотек (по стандарту "requirements.txt")"
*флаг -r указывает на то, чтобы установить не сам файл списка,
 а файлы (библиотеки), указанные в файле (списке).
3.Можно использовать:
$pip freeze > "любое название".txt
*создаёт список библиотек из установленных на локальном репозитории;
 используется для того, чтобы потом можно было через него установить
 все ранее использовавшиеся библиотеки и чтобы не приходилось каждый раз
 устанавливать относительно объёмный перечень всех бибилиотек.
